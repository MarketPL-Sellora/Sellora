# Test Plan: Платформа групових закупівель «Sellora»

## 1. Вступ (Introduction)
**Мета:** Гарантувати стабільність, безпеку та коректність бізнес-логіки колективних покупок, автоматизації логістики та розмежування прав доступу між користувачами.

**Об’єкт тестування:**
* **Module 1: Group Buy Engine** — логіка створення кімнат, холдування коштів та таймери (24г).
* **Module 2: Product Discovery** — повнотекстовий пошук та складна фільтрація товарів.
* **Module 3: Identity Management** — реєстрація та налаштування профілів (RBAC: Buyer/Seller).
* **Module 4: Order Lifecycle** — життєвий цикл замовлення та фінансові транзакції.
* **Module 5: Logistics Integration** — автоматизація ТТН через API (Нова Пошта/Укрпошта).
* **Module 6: Notification Engine** — система Push, Email та Telegram сповіщень.

---

## 2. Стратегія тестування (Testing Strategy)

### 2.1. Рівні тестування
* **Unit Testing:** Перевірка атомарних функцій (валідація ІПН, розрахунок знижки, обрізка довгих рядків).
* **Integration Testing:** Перевірка взаємодії модулів: `Payment Gateway` ↔ `Order Status`, `Logic API` ↔ `Logistic API`.
* **System Testing (E2E):** Проходження повного користувацького шляху: від реєстрації продавця до отримання товару покупцем.

### 2.2. Види тестування
* **Functional:** Перевірка відповідності системи вимогам (чи закривається група при 3/3 учасниках).
* **Security:** Тестування на вразливості (SQL Injection, XSS, маніпуляція ціною в API запросах).
* **Boundary Value Analysis (BVA):** Тестування межових значень (вага 0.01 кг, 1 учасник, пароль на мін/макс символів).
* **API Testing:** Тестування бекенду без GUI через Postman (перевірка статус-кодів 200, 201, 400, 403, 500).
* **UI/UX:** Перевірка адаптивності під мобільні пристрої та візуального фідбеку (спінери, повідомлення про помилки).

## 3. Ресурси та інструменти (Tools)
* **API:** Postman / Swagger UI.
* **Backend Testing:** JUnit (Java) або PyTest (Python).
* **Manual Testing:** Chrome DevTools (Network, Console, Device Toolbar).
* **Bug Tracking:** GitHub Issues або Notion Board.
* **Environment:** Локальне середовище розробки, Sandbox API Нової Пошти.

---

## 4. Критерії «Прийняття» (Acceptance Criteria)

| Критерій | Опис |
| :--- | :--- |
| **Критерії початку** | Код розгорнуто, БД заповнена тестовими даними, API-ключі активні. |
| **Критерії завершення** | 100% тест-кейсів виконано. Відсутні баги рівнів **Critical** та **High**. |
| **Якість коду** | Unit-тести покривають мінімум 70% бізнес-логіки. |

---

## 5. План виконання (Schedule)

1.  **Фаза 1 (Тиждень 1):** Тестування авторизації, безпеки профілів та пошукової системи.
2.  **Фаза 2 (Тиждень 2):** Тестування ядра системи (Group Buy) та логіки списання коштів.
3.  **Фаза 3 (Тиждень 3):** Інтеграція з логістикою, генерація ТТН та система сповіщень.
4.  **Фаза 4 (Тиждень 4):** Регресійне тестування, UI/UX аудит та підготовка звіту.
