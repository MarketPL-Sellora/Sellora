# Специфікація форматів обміну даними (Data Contracts)

## 1. Опис моделей даних (DTO Specification)
Усі контракти перевірено та збережено у форматі JSON у папці `docs/contracts/` репозиторію. У відповідях бекенду відсутні чутливі дані (хеші паролів) та внутрішні ключі БД (наприклад, `__v` або `sys_id`).
* **Авторизація:** [login-contract.json](./contracts/login-contract.json)
* **Створення замовлення:** [order-contract.json](./contracts/order-contract.json)
* **Групові покупки:** [group-buy-contract.json](./contracts/group-buy-contract.json)

## 2. Глобальна схема помилок
Для уникнення падінь фронтенду при кожній помилці, сервер Sellora завжди повертає уніфікований об'єкт помилки (містить `timestamp`, `errorCode`, `message`, `path` та опціональний масив `details`).
* **Приклад схеми:** [global-error.json](./contracts/global-error.json)

## 3. Мапа статус-кодів (Status Code Map)
Використання HTTP-кодів у бізнес-реаліях Sellora:

| Код | Назва | Коли використовується у проєкті Sellora (Конкретний бізнес-кейс) |
| :--- | :--- | :--- |
| **200** | OK | Успішне отримання історії замовлень користувача або оновлення його профілю доставки. |
| **201** | Created | Оформлено нове замовлення, система згенерувала номер накладної та посилання на оплату (LiqPay). |
| **400** | Bad Request | Фронтенд надіслав невалідні дані: пароль при реєстрації коротший за 8 символів або вказана від'ємна кількість товару. |
| **401** | Unauthorized | Користувач намагається відкрити "Кошик", але термін дії його JWT-токена минув (або токен відсутній). |
| **403** | Forbidden | Авторизований покупець намагається надіслати запит на зміну ціни товару (ця дія дозволена лише для Admin). |
| **404** | Not Found | Покупець перейшов за посиланням на "Групову покупку", термін дії якої вже вичерпано, або ввів ID товару, якого не існує. |
| **409** | Conflict | Спроба зареєструвати акаунт на Email, який вже є в базі, АБО спроба замовити 5 одиниць товару, тоді як на складі залишилося лише 2. |
| **500** | Internal Server Error | Неочікувана відмова сервера: втрачено з'єднання з PostgreSQL під час збереження транзакції. |
