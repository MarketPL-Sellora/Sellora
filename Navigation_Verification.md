## Чек-лист перевірки ключових маршрутів:

### 1. Головна навігація (Header / Footer)
- [x] Клік на логотип завжди повертає на Головну сторінку (`/`).
- [x] Посилання на "Каталог" коректно відкриває сторінку списку товарів.
- [x] Іконка Кошика відкриває модальне вікно або переводить на сторінку кошика.
- [x] Посилання в Footer (Умови використання, Контакти тощо) ведуть на відповідні існуючі сторінки або мають коректні заглушки.

### 2. Життєвий цикл товару (Catalog -> Product Detail)
- [x] Клік на картку товару в каталозі (`ProductGrid`) успішно переводить на сторінку деталей цього товару (`/product/:id`).
- [x] Кнопка "Назад" у браузері коректно повертає зі сторінки товару назад у Каталог із збереженням попередньої сторінки пагінації.

### 3. Навігація особистого кабінету (Cabinet)
- [x] Редірект неавторизованого користувача: при спробі зайти на `/cabinet` без токена, користувача перекидає на сторінку Логіну.
- [x] Усередині кабінету вкладки (Реквізити, Замовлення, Налаштування) перемикаються без помилок у консолі.

### 4. Обробка помилок (Error Handling)
- [x] **Сторінка 404 (Not Found):** При введенні неіснуючої URL-адреси (наприклад, `/some-random-url`) користувач бачить кастомну сторінку 404 з кнопкою "Повернутися на головну".
- [x] Неіснуючий товар: при спробі відкрити `/product/9999999` (якщо бекенд повертає помилку 404), користувач бачить відповідне повідомлення, а не порожній екран.

**Висновок:** Навігаційна карта відповідає вимогам. "Битих" посилань або цикли нескінченних редіректів не виявлено.
