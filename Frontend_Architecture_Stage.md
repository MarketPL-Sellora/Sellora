# Опис архітектури Frontend-додатка та глобального стану (Stage 5)

## 1. Загальна архітектура та технологічний стек
Frontend-частина побудована за принципами компонентного підходу з використанням сучасного стеку технологій:
* **Фреймворк:** Vue 3 (Composition API)
* **Збирач:** Vite (для швидкої розробки та оптимізованої збірки)
* **Управління станом:** Pinia
* **Стилізація:** Tailwind CSS
* **Взаємодія з API:** Axios (з налаштованими інтерсепторами для обробки помилок та токенів)

## 2. Структура директорій (Folder Structure)
Проєкт має чітку ієрархію, яка розділяє логіку, представлення та конфігурації. У рамках "Zero Mocks Policy" всі хардкод-тексти винесені в окремі константи.

```text
src/
├── assets/             # Статичні файли (зображення, іконки, глобальні стилі)
├── components/         # Перевикористовувані Vue-компоненти
│   ├── product/        # Компоненти товарів (ProductGrid, ProductCard)
│   ├── forms/          # Форми з валідацією (CreateStoreForm)
│   ├── modals/         # Модальні вікна (CartModal з логікою перевірки залишків)
│   └── cabinet/        # Компоненти особистого кабінету (CabinetRequisites)
├── constants/          # Глобальні константи
│   └── dictionary.ts   # Словник текстів UI (Zero Mocks Policy)
├── router/             # Налаштування Vue Router (картинг сторінок, захист маршрутів)
├── state/              # Управління глобальним станом (Pinia)
│   ├── productStore.ts # Стан каталогу, пагінація, сортування
│   ├── cartStore.ts    # Стан кошика, підрахунок сум
│   └── authStore.ts    # Стан авторизації та профілю користувача
├── views/              # Головні сторінки (Home, Catalog, ProductDetail, Checkout, Cabinet)
├── types/              # TypeScript інтерфейси (спільні DTO-контракти з Backend)
└── App.vue             # Кореневий компонент
